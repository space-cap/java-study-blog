배열(Array)
배열은 같은 타입의 데이터를 연속적으로 저장하는 자료구조입니다. 
배열은 메모리에서 연속된 공간을 차지하며, 각 요소는 인덱스를 통해 접근할 수 있습니다. 배열은 고정된 크기를 가지며, 크기를 변경할 수 없습니다. 
배열의 크기는 선언 시에 지정해야 합니다.

배열은 다음과 같은 특징을 가지고 있습니다.
1. 고정 크기: 배열은 선언 시에 크기를 지정해야 하며, 이후에는 크기를 변경할 수 없습니다.
2. 연속된 메모리 공간: 배열은 메모리에서 연속된 공간을 차지합니다. 각 요소는 인덱스를 통해 접근할 수 있습니다.
3. 동일한 타입: 배열은 동일한 타입의 데이터만 저장할 수 있습니다. 서로 다른 타입의 데이터를 저장할 수 없습니다.
4. 빠른 접근 속도: 배열은 인덱스를 통해 직접 접근할 수 있으므로, 데이터에 대한 접근 속도가 빠릅니다.
5. 메모리 낭비: 배열은 고정된 크기를 가지므로, 필요 이상의 메모리를 할당할 수 있습니다. 이로 인해 메모리 낭비가 발생할 수 있습니다.
